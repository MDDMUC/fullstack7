# Ticket: UI/UX Polish

ID: TICKET-UX-002
Owner: Product/Design/Eng
Status: Proposed
Priority: P1
Created: 2025-12-22
Last updated: 2025-12-22
Target window: Pre-launch
Related docs: `PROJECT_CONTEXT.md`

## Problem
Small UI/UX issues can hurt perceived quality and activation during the first cohort.

## Goal
Complete a focused polish pass to remove friction and improve clarity on mobile.

## Non-goals
- No new major features.
- No visual redesign or rebranding.

## Scope
- Review core flows: onboarding, home, chats, events, crews, gyms.
- Fix minor layout, spacing, and readability issues.
- Improve microcopy clarity where users hesitate.
- Validate loading and empty states.

## Requirements
- Mobile-first (iOS Safari, Android Chrome).
- Use design tokens only.
- No new pages without MobileTopbar and MobileNavbar.

## Success metrics
- Reduced user confusion in test cohort feedback.
- No new UI regressions in QA.

## Dependencies
- None.

## Definition of Done
- Targeted polish items resolved and verified on mobile.
